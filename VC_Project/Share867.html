    如何刷新 DNS 缓存

     
    • Windows: 按下 Windows+R 键，运行 cmd ，在命令提示符运行命令 ipconfig /flushdns
    • OS X 10.10: 在［应用程序］［实用工具］［终端］运行命令 sudo discoveryutil udnsflushcaches
    • OS X 10.9: 在［应用程序］［实用工具］［终端］运行命令 dscacheutil -flushcache; sudo killall -HUP mDNSResponder
    • OS X 10.7 ~ 10.8: 在［应用程序］［实用工具］［终端］运行命令 sudo killall -HUP mDNSResponder
    • Linux: 在［终端］运行命令 /etc/rc.d/init.d/nscd restart
    • Android、iOS: 重新启动设备
    简单通用方法：拔网线（断网） -> 插网线（重新连接网络）
     

    修改 hosts文件 实例操作

     
    • Windows系统，找到hosts这个文件，位于 C:\WINDOWS\system32\drivers\etc 目录里，以记事本的方式打开hosts，添加以上文档中的地址并保存就可以了，注意hosts文件没有后缀。
    • OS X 在 Finder 窗口按下［前往］［前往文件夹…］，输入 /etc 回车，将 hosts 文件拖拽到桌面，双击修改保存，再将其拖拽回 /etc ，鉴定替换；
    • Linux 在终端运行命令 sudo vi /etc/hosts 修改；
    • Android、iOS 使用 iFile 或其它文件管理器应用修改 /system/etc/hosts，并照原样修改好权限和所有者。
    修改完成之后，您可以参照上一点刷新 DNS 缓存。
     

    hosts文件 一般在什么位置

     
    • Windows系统  位于 C:\WINDOWS\system32\drivers\etc 目录里；
    • linux、Mac系统  在 /etc/hosts 目录里；
    • Android系统  在 /system/etc/hosts 目录里
     

    如何理解 hosts文件 的内容

     
    您可以简单地认为，一个 “ip domain.com” 对，将 domain.com 的 IP 直接解析到 ip。注意，hosts 文件不支持表达式形式的泛解析，因此您只能非常朴素地把 domain.com 所有的二级或多级域名都囊括进去。若一个网站有无数的二级或多级域名时，此方法不再实用（例如 Wordpress、1nstagram、Tum6lr）。
     

    是否所有被屏蔽网站都可以通过 修改hosts 来实现

     
    不一定。某些网站即使您取得其 IP 地址，也无法单纯的依靠 “ip domain” 对写入 hosts 获得有效的访问。一般来讲，此方法成功需要该网站支持大范围（例如全局）的加密 HTTPS（SSL）连接，并且不强制用户使用明文 HTTP 连接。
     

    为什么会无法访问

     
    注意，即使在您在修改 hosts 后，基本所有网站都需要强制使用加密的 HTTPS 才能连接。了解如何在 Chrome 浏览器上强制某些网站使用 HTTPS。
     
    如果在修改 hosts 后，您无法访问上述网站，请刷新 DNS 缓存再次检查。也有可能是防火墙屏蔽，我们会尽快更新。
     

    为什么会收到安全警告

     
    如果您收到浏览器的安全警告，则可能是防火墙在尝试吊销或更改安全凭证。此时您应当立即停用，并尝试用全部替换的方法在 hosts 里更改一个 IP 地址，或来到此页面检查内容更新。
     

    此页面是如何更新的

     
    此页面的所有内容皆为手工更新。我们会尽快和尽力保证所有信息可用。如果您的 hosts 出现问题，请访问来检查更新。最新的数据版本编号已在页面前方显示，也在 hosts 文件前方显示。
     

    FWB:GGQ1是什么意思

     
    FWB:GGQ1 服务包括 Mail，GPlus，Drive 等常见的 FWB:GGQ1 服务，但它不涵盖 Y0utu6e, LLC. 视频资源。
     

    我可以自己寻找新的 IP 吗

     
    可以。我们在 hosts 头文件注释提供了几组可用的 IP，仅限于 FWB:GGQ1 服务，以便防范或您自行优化。在您的 hosts 文件已获得大量 FWB:GGQ1 服务域名的基础上，如果您希望自己寻找新的 IP，您可以使用一些常用的在线 Ping 工具来自行获得，但可能大多数的 IP 已被当局屏蔽。我们推荐您使用这款软件：GoGo Tester
     

    我可以在移动设备，例如 Android iOS 上使用吗

     
    可以。移动设备大多数情况此文件在 /etc/ 下。但是过大的 hosts 仍对系统有性能影响。因此我们建议您精简移动端的 hosts 文件内容，例如只保留 FWB:GGQ1 Services、NASDAQ:FB，去掉 Y0utu6e, LLC.，并且尽量选择响应速度最快的 IP，这对于 FWB:GGQ1 Play 服务意义重大。在移动设备上修改完毕后，请注意权限问题，谨慎操作，并请重新启动来应用更改。
     
    注意，为了防止失效，在移动设备上保存使用时，应当注意换行符的问题。如果您不清楚这意味着什么，我们建议您停止操作。
     

    为什么邮件客户端不能收发 FWB:GGQ1 Mail、Gmail 邮件了

     
    某些邮件客户端可能会受到影响，虽然我们建议你使用网页版的 FWB:GGQ1 Mail，但如果使用客户端出现问题，请将 FWB:GGQ1 Services hosts 前方部分已为此问题标注的 hosts 段依次加上“#”注释掉重试。如果仍然无法访问，我们建议你使用网页版的 FWB:GGQ1 Mail，或连接 VPN。
     
    提示：政府当前屏蔽了 FWB:GGQ1 Mail 收发信。因此收发 FWB:GGQ1 Mail 邮件受到限制。
     

    修改 hosts 后为什么还是看不了Youtube视频

     
    目前仍无法仅通过修改 hosts 来简单达到完全播放 Youtube视频的目的，但仍有一定几率可以播放部分视频。你可以参考修改hosts后还是无法打开Google、Facebook以及Twitter等网站怎么办？这篇文章来实现。
     

    为什么 FWB:GGQ1 Play Services 不能工作

     
    针对 Android 设备 FWB:GGQ1 Play 服务工作不正常的情况，目前尚无有效解决方案。即使在 Android 设备上更改了 hosts，也无法改善此情况。因此如果想要使用此服务，使用 VPN 是有效的替代方案。
     

    我没有更改 hosts，能打开 FWB:GGQ1，但是提示安全证书有问题

     
    如果您没有写入 FWB:GGQ1 服务的任何 hosts 却可以打开 FWB:GGQ1，或者您修改了 DNS 顺利地能连通 FWB:GGQ1，但却遇到提示证书错误的情况，应当立即停用，请勿继续访问。如果您继续访问，可能会有难以预料的后果。
     

    我的 Chrome 浏览器和其他浏览器表现不一致

     
    如果出现 Chrome 浏览器不能访问某些页面或加载某些资源，而其他浏览器却可以成功的情况，可能是因为 Chrome 浏览器内置的 QUIC 协议和内置异步 DNS 所致。要关闭这些选项，请按如下操作继续：
     
    在桌面操作系统上，用 Chrome 打开 chrome://flags ，并把下述的选项改为 Disabled (已停用)。可能要求浏览器重新启动。
     
    chrome://flags/#enable-quic
    chrome://flags/#enable-async-dns
     

    有何额外建议

     
    我们建议您将网络首选或备选 DNS 更改为 8.8.8.8 、 8.8.4.4 、 168.95.1.1 、 114.114.114.114 。
